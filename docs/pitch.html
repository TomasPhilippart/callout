Callout

An ambient voice interface for developers running AI agents. Speaks up when your agents need you. Listens when you respond.

macOS · Linux · Rust · on-device

The problem

AI agents do the work. But you still end up babysitting — watching terminals, switching windows, checking if something needs a decision. The more agents you run, the worse it gets.

Callout removes the watching. You work, cook, or do whatever — and it speaks up the moment a decision is needed. You answer out loud. It routes the answer back. Done.

How it works

1
Agent hits the Callout API Any agent, any language, any framework — a single POST request.
2
Callout speaks the question out loud "auth-refactor asks: Should I delete the generated files? A: Yes. B: No."
3
You respond by voice Hold the push-to-talk hotkey, say your answer, release.
4
The agent gets the answer and continues The HTTP request resolves. No polling, no webhooks, no UI to open.

You can also ask what's happening at any time — press the hotkey, say "what's running?" and Callout tells you in plain language.

API

A local HTTP server on localhost:7878. Three endpoints. Works from curl, any language, any agent framework.

POST
/agents/register Register an agent with a name and optional context terms. Returns an agent_id for subsequent calls.
POST
/notify Fire-and-forget. Callout speaks the message and logs it. Returns immediately.
POST
/ask Blocking. Speaks the question, listens for your response, returns the answer. Supports multiple-choice options, multi-select, and timeout with a default.
GET
/status Returns all registered agents, their state, and last-seen time.
DELETE
/agents/{id} Deregister an agent. Agents inactive for 5 minutes are marked stale automatically. 
curl -X POST localhost:7878/ask \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "a1b2c3",
    "question": "Which approach should I take?",
    "choices": [
      { "key": "A", "label": "Rewrite from scratch" },
      { "key": "B", "label": "Patch the existing code" }
    ],
    "timeout_seconds": 120,
    "default": "B"
  }'

→ { "answer": "B", "answers": ["B"], "raw": "patch it", "timed_out": false }

Voice

Speech-to-text

Whisper.cpp via whisper-rs. On-device, no API keys. Model lives in ~/.callout/models/. Tiny model is fast enough; base model is the default.

Text-to-speech

macOS: built-in say command (Apple neural voices). Linux: Piper TTS with a local voice model. Zero cloud dependency.

Push-to-talk

Global hotkey (default: Option / Alt). Hold to record, release to transcribe. Works across all apps, no focus required.

Glossary

Domain terms in ~/.callout/glossary.toml fed to Whisper as context. Fixes "Cloud" → "Claude", project-specific names, etc. Agents add their own terms at registration.

Menu bar

Four icon states — idle, active, waiting (agent blocked on /ask), listening (hotkey held). Click to see active agents and a recent log of messages.
